```bash
# Using constants

# Bitwise operations
cargo run -- -B "12 & 10"          # Output: 8
cargo run -- -Bx "0xFF & 0x0F"     # Output: 0xF

# Format conversion
cargo run -- -x "255"              # Output: 0xFF
cargo run -- -o "255"              # Output: 0o377
cargo run -- -b "255"              # Output: 0b11111111

# Scientific notation (shows both formats)
cargo run -- "1e3 * 2"             # Output: 2000
                                           #   (scientific: 2.000000e3)

# Negative non-decimal numbers (use parentheses)
cargo run -- "-(0xFF)"             # Output: -255
cargo run -- "-0xFF"               # Error: Negative non-decimal numbers are not supported

# View help
cargo run -- -F
cargo run -- operators
cargo run -- --help
```

## FAQ

**Q: Why use i64 for bitwise operations?**
A: i64 provides sufficient bit width while supporting negative number bitwise operations.

**Q: How to add a new function?**
A:
1. Implement the function in `functions.rs`
2. Add the function name to `FUNCTIONS` array in `operator.rs`
3. Add handling logic in `eval_postfix()` in `evaluator.rs`
4. Add documentation in `repl.rs` help information

**Q: What's the difference between `=` and `==`?**
A:
- `=` is an equality check that returns `yes` or `no`
- `==` is a logical equality that returns `true` or `false`
- Both compare if two values are equal, but with different output styles

**Q: Why do `<` and `>` return `true`/`false` but `=` returns `yes`/`no`?**
A: This design distinguishes relational operators (`<`, `>`, `==`) that return boolean values from the equality check operator (`=`) that returns yes/no for simple equality verification.

**Q: Why are negative non-decimal numbers not supported?**
A: To avoid ambiguity between unary minus and negative literals. Use parentheses: `-(0xFF)` instead of `-0xFF`.

**Q: How to define a user constant?**
A: Use `create const NAME value` in interactive mode. Note: Names starting with `C_` are reserved.

**Q: How does scientific notation output work?**
A: When input contains scientific notation (e.g., `1e3`), output shows both regular and scientific formats.
